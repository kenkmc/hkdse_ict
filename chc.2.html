2.1 傳輸控制協定 / 互聯網協定 (TCP/IP)

解釋數據傳輸的原理：數據如何在網絡中被拆分與重組。

點擊「開始傳輸」觀察 TCP 與 IP 如何分工合作。

傳送端 (TCP)

接收端 (TCP)

1. 傳輸控制協定 (TCP)

負責將大型數據拆分成多個封包 (Packets)並編號；在接收端重新組合封包，確保數據準確無誤。

2. 互聯網協定 (IP)

負責為每個封包加上目的地 IP 位址，並透過路由器尋找最佳傳輸路徑送達目的地。

2.2 互聯網協定 (IP) 位址

32 位元 (32-bit)

IPv4 格式

192.168.1.254

由四組 0-255 的數字組成。因互聯網普及，IPv4 地址已不敷使用。

128 位元 (128-bit)

IPv6 格式

2001:0DB8:AC10:FE01:0000:0000:0000:0001

使用十六進制表示，提供近乎無限的位址數量以解決 IPv4 枯竭問題。

2.4 劃一資源定位符 (URL)

URL (Uniform Resource Locator) 就是我們常說的「網址」，用來指示網頁或檔案在互聯網上的確切位置。完整的 URL 包含以下四個部分：

https:// www.school.edu.hk :443 /index.html
1. 通訊協定

指定使用的資料傳輸規則（如 HTTP/HTTPS 或 FTP）。

2. 網域名稱

識別互聯網上寄存該網頁的伺服器名稱。

3. 連接埠 (Port)

指定伺服器提供服務的特定通道（HTTPS 預設為 443）。

4. 檔案路徑

資源在伺服器上的特定資料夾與檔案名稱。

2.5 常見網絡協定 (Network Protocols)

HTTP 與 HTTPS 的分別 (加密概念)

Hypertext Transfer Protocol (Secure)

HTTP (明文傳輸)

數據未經加密。若在傳輸過程中被黑客攔截，他們可以直接讀取你的敏感資料（如密碼）。

PWD:123
攔截: PWD:123

HTTPS (加密傳輸)

傳送前會將數據進行加密 (Encryption) 變成亂碼。黑客即使攔截也無法解讀，到達伺服器後才會解密。

無法解讀: !#@%&*

SMTP / POP3 / IMAP

Email Protocols

電子郵件的運作依賴不同協定分工：
• SMTP：負責將郵件發送到伺服器。
• POP3 / IMAP：負責從郵件伺服器接收/下載電子郵件到使用者的裝置。

郵件伺服器

FTP

File Transfer Protocol

檔案傳輸協定。常用於客戶端與伺服器之間上載或下載大型檔案（例如網頁管理員上傳網站檔案）。

HTTP 狀態信息 (Status Codes)

當瀏覽器請求網頁時，伺服器會傳回狀態碼：

200 OK
請求成功，網頁正常載入。
404 Not Found
客戶端錯誤，找不到請求的網頁。
500 Error
伺服器發生內部意外錯誤。
